Задание №1

Что будет выведено в консоль ?

var text = 'outside';
function logIt(){
console.log(text);
var text = 'inside';
};
logIt();
Узнать ответ
На экран будет выведено: "undefined" !
Т.к сначала пробигается по коду compiler, он доходит до функции console.log видит, переменную text, находит её внутри функции, но поскольку присвоение этой переменной происходит после console.log, то на момент вывода в консоль эта переменная не имеет значение.

Задание №2

Что будет выведено в консоль ?

(function() {
var a = b = 5;
})();
Узнать ответ
В консоль не выведется ничего, т.к. нету функции вывода.
В результате кода, переменной а и b будет присвоено значение 5. При этом b - будет глобальной переменной.

Задание №3

Что будет выведено в консоль ?

function test() {
console.log(a);
console.log(foo());
var a = 1;
function foo() {
return 2;
}
}
test();
Узнать ответ
console.log(a); - выведет undefined;
console.log(foo()); - выведет 2
Так произойдет потому, что функции имеют приоритет выше чем переменные. Compiler перетаскивает наверх (образно) кода только объвление переменной, а функцию - полностью.

Задание №4

Что будет выведено в консоль ?

myname = "global";
function func() {
console.log(myname);
var myname = "local";
console.log(myname);
}
func();
Узнать ответ
В этом задании результат будет абсолютно таким же как и в первом задании. При первом выводе будет выведено "undefined" при втором выводе будет выведено "local" т.к. сначала объявление, а потом уже присвоение + функция имеет свой scope, по этому до глобальной переменной мы не доходим.

Задания №5,6,7

5. С помощью какой конструкции языка можно управлять потоком выполнения скрипта и отслеживать ошибки?
6. Что такое глобальные переменные ? Как они создаются ? Какие проблемы связаны с использованием глобальных переменных?
7. Что такое замыкания (closure) в JavaScript?
Узнать ответ
5. В Java Script можно управлять потоком выполнения скрипта и отслеживать ошибки с помощью конструкции try...catch.
6. Глобальными называют переменные и функции, которые не находятся внутри какой-то функции. То есть, иными словами, если переменная или функция не находятся внутри конструкции function, то они – «глобальные». Проблема, которая связанна с глобальными переменными это - shadowing.
7. Замыкание – это функция вместе со всеми внешними переменными, которые ей доступны, но обычно, говоря «замыкание функции», подразумевают не саму эту функцию, а именно внешние переменные.

Задание №8

Какая разница между двумя способами
задания функции ?

var foo = function(){
// Some code
};
function bar(){
// Some code
};
Узнать ответ
В первом способе, мы объвляем переменную, и присваиваем ей анонимную функцию, обратится к этой функции мы можем так: foo();
Во втором способе, мы объявляем обычную именнованую функцию, обратится к функции можем так: bar();

Задание №9

Что будет выведено в консоль ?

(function() {
baz = 5;
var bar = 10;
})();
console.log(baz);
console.log(bar);
Узнать ответ
console.log(baz); - будет выведено "5", т.к. в результате выполнения функции эта переменная станет глобальной.
console.log(bar); - будет выведено "Uncaught ReferenceError: bar is not defined" т.к. такой переменной в Lexical scope - нету.

Задание №10

Что будет выведено в консоль ?

(function() {
var foo = 1;
function bar() {
var foo = 2;
}
bar();
console.log(foo);
if(true) {
var foo = 3;
}
console.log(foo);
})();
Узнать ответ
При первом выводе console.log(foo); - будет выведено 1, 2 - выведено не будет, т.к. у функции bar() свой личный scope.
При втором выводе console.log(foo); - будет выведено 3. т.к. условный оператор if сработает и по скольку он не имеет своейго scope, он перезапишет глобальную переменную foo.
